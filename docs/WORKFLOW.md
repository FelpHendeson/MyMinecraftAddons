# Fluxo Geral de Trabalho

Este é o fluxo base para desenvolver, testar e repetir alterações em addons de Minecraft Bedrock.

## Ciclo de desenvolvimento

1. Desenvolver o addon no PC.
2. Exportar o projeto como `.mcpack` ou `.mcaddon`, conforme a necessidade.
3. Enviar o arquivo exportado para o celular.
4. Abrir o arquivo com Minecraft Bedrock.
5. Ativar o behavior pack e o resource pack no mundo de teste.
6. Testar o comportamento dentro do jogo.
7. Corrigir problemas encontrados.
8. Repetir o ciclo até o resultado esperado.

## Boas práticas

- Testar sempre em um mundo separado de desenvolvimento.
- Fazer mudanças pequenas e verificáveis.
- Registrar alterações relevantes no changelog do addon.
- Não misturar arquivos de projetos diferentes.
- Validar nomes, namespaces e estrutura antes de empacotar.
