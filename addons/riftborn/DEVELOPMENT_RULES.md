# Regras de Desenvolvimento

Estas regras guiam a evolução do Riftborn.

## Regras gerais

- Desenvolver de forma incremental.
- Manter cada versão pequena, testável e documentada.
- Não implementar sistemas complexos antes da base técnica.
- Usar o namespace planejado `riftborn` quando arquivos funcionais forem criados futuramente.
- Separar corretamente behavior pack, resource pack, documentação, assets e distribuição.
- Não criar conteúdo fora do escopo solicitado.
- Não misturar contexto de outros addons.

## Nesta etapa

Nesta etapa, somente estrutura de pastas e documentação devem existir.

Não criar:

- `manifest.json`.
- Itens.
- Receitas.
- Texturas.
- Scripts.
- Mobs.
- Loot tables.
- Funções.
- Arquivos funcionais de addon.

## Escopo inicial futuro

Quando o desenvolvimento funcional começar, a primeira fase poderá incluir:

- Base técnica.
- Documentação técnica.
- Manifests.
- Itens simples.
- Receitas simples.
- Empacotamento.

Esses itens só devem ser criados quando forem explicitamente solicitados.
